# AGENTS.md - docs

## Mission
Keep Footie's operating system, architecture, plans, memory, PR drafts, and review notes durable, concise, and actionable for autonomous development.

## Build Order
1. Core governance docs.
2. Durable memory.
3. Architecture and milestone plans.
4. Progress, review, and PR artifacts.

## Autonomy Rules
- Update docs whenever durable decisions or blockers change.
- Prefer concise, durable notes over verbose historical noise.

## Coding Rules
- Use clear headings, bullet structure, and explicit status/risk statements.

## Testing Rules
- Validate docs by keeping referenced paths/commands accurate.

## Data Contract Rules
- Docs must distinguish planned vs implemented contracts.

## Schema / Migration Rules
- Any schema guidance must describe safety and rollback considerations.

## Logging Rules
- Progress logs should note date, status, checks, and blockers.

## PR Rules
- Maintain local PR drafts when remote PR creation is unavailable.

## Memory Maintenance Rules
- Keep `MEMORY.md` compact and move detail into topic files.

## Publish Gate Rules
- Do not describe unvalidated pipelines as production-ready.
